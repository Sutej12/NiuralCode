Key Data Flows

1

Application Flow

  • Candidate fills form on Career Page
  • Validate: duplicate, file type, size, referral, job status, hired check
  • Save candidate + resume to database
  • Background thread sends Gmail confirmation
  • Email includes HMAC portal tracking link
2

AI Screening Flow

  • Admin clicks "Screen" on dashboard
  • Extract text from PDF/DOCX (PyPDF2, python-docx)
  • Send to OpenAI with job description + requirements
  • Receive: score, rationale, parsed skills
  • Auto-advance: Applied → Screened → Shortlisted
3

Scheduling Flow

  • Query Google Calendar freebusy for open slots
  • Block 5 tentative events (prevents conflicts)
  • Email candidate with selection link
  • Candidate selects → confirm + generate Meet link
  • Release other holds, send .ics invite to both
4

Interview + Analysis Flow

  • Web Speech API captures live speech
  • Speaker toggle labels Interviewer / Candidate
  • Lines appended with timestamps to database
  • Finalize saves permanent transcript
  • AI analysis: ratings, strengths, hire recommendation
5

Offer → Onboard Flow

  • OpenAI generates HTML offer letter
  • Admin sends via email to candidate
  • Candidate signs with canvas signature (base64)
  • Status: Hired → auto-trigger Slack invite
  • team_join webhook → welcome msg → Onboarded

Critical Edge Cases Handled

🔒 Duplicate Applications

Database unique_together(email, job) prevents duplicates at the DB level. Serializer validate() catches it earlier with a friendly message: "You have already applied for this position." Candidates can still apply to different roles.

📄 File Validation

5 layers: extension check (PDF/DOCX only), content-type check, file size limit (5MB), Django server limits (10MB), and frontend client-side validation for instant feedback.

⏸️ Role Closed / Paused

validate_job() blocks submissions to non-Open roles. Career page hides closed jobs. On close/pause, all active candidates receive notification emails. Admin can reopen roles at any time.

📅 Slot Conflict Prevention

Tentative calendar holds block slots from the moment they are offered. DB UniqueConstraint on (interviewer_email, start_time) for confirmed slots. Atomic release of other holds when one is confirmed. Google Calendar freebusy as source of truth.

🚫 Hired / Onboarded Block

If a candidate is already Hired or Onboarded for any role, they cannot apply to new positions. Returns: "You are already hired / onboarded. Contact HR for role changes." Terminal states (Hired, Onboarded, Rejected) are locked from further status changes.

⏰ Unresponsive Candidates

Follow-up email button on scheduling page sends reminders with available slots. Tracks follow_up_sent flag. Reschedule flow with interviewer approve/reject via bell notifications. Admin can manually release stale slots.
